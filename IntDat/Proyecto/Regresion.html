Como se coment en el apartado de EDA:

Se nos pide elegir 5 regresores para la regresin y contamos exactamente con ese nmero, por lo que no podemos descartar ninguna variable. An as, hemos visto que tenemos algunas variables ms interesentas que otras. Varibles correladas con la salida nos aumentan las posibilidades de obtener un buen regresor, pero debemos evitar usar variables correladas entre s para evitar la multicolinealidad. Sera conveniente evitarla para aumentar la interpretabilidad del modelo, pero la potencia en s de este no cambia.

Graficamos la relacin de cada variable respecto a la salida

ggplot(melt(auto, "Mpg"), aes(x=value, y=Mpg, color=variable)) +
  geom_point(alpha=0.3) +
  facet_wrap(.~variable, scale="free") +
  theme_light()

Como dijimos, se aprecia alta correlacin entre Displacement, Horse_power, Weight respecto de la salida, probablemente de forma logartmica.

Las matrices nos correlacin nos confirman esta idea (con coeficientes de Pearson y Kendall)

corrplot.mixed(cor(auto), tl.pos="lt", upper="color", title="Pearson")

corrplot.mixed(cor(auto, method="kendall"), tl.pos="lt", upper="color", title="Kendall")

Por tando, si las ordenramos por cules parecen ser ms prometedoras, tendramos: Weight > Displacement > Horse_power > Model_year > Acceleration

Tambin tenemos que tener en cuenta que las tres primeras variables estn correladas entre s.

run_lm_fold <- function(i, x, tt = "test") {
    file <- paste(x, "-5-", i, "tra.dat", sep="")
    x_tra <- read.csv(file, comment.char="@", header=FALSE)
    file <- paste(x, "-5-", i, "tst.dat", sep="")
    x_tst <- read.csv(file, comment.char="@", header=FALSE)
    In <- length(names(x_tra)) - 1
    
    names(x_tra)[1:In] <- paste ("X", 1:In, sep="")
    names(x_tra)[In+1] <- "Y"
    names(x_tst)[1:In] <- paste ("X", 1:In, sep="")
    names(x_tst)[In+1] <- "Y"
    
    if (tt == "train") {
        test <- x_tra
    }
    else {
        test <- x_tst
    }

}

Con el proceso de cross-validation dividimos el dataset en N subconjuntos (folds) y repetimos el entrenamiento N veces. Cada entrenamiento se aplica reservando uno de los subconjuntos como test y entrenando con el resto. Al final, el error obtenido para el modelo es la media de los errores en cada fold. La eleccin del nmero de folds es importante y si el problema lo permite (en trminos de gasto computacional), se debera probar con varios. En este caso hemos utilizado 5 folds.

Con esto conseguimos no desperdiciar el conocimiento del conjunto de test y no guiarnos por una nica evaluacin del modelo.

Como resultados, nos muestra que los modelos con los que obtuvimos mejores resultados de R2 y RSME en sus apartados han acabado con mejor RSME tras el cross-validation. Tambin apreciamos que con KNN conseguimos ligeramente mejores resultados.
